##### 제목(title) ####
# 50자 이내로 변경사항을 작성 / 제목과 본문은 한 줄 띄워 분리
# <Type>: <Title>
# ex) docs: Add new post


##### 본문(content) #####
# 구체적인 내용을 작성
# 여러 줄인 경우 '-'로 구분 (한 줄은 72자 이내)

##### 꼬리말(footer) #####
# 깃허브는 이슈종료 예약어 있음
# 현재 커밋과 관련된 이슈 번호 추가 등
# <Type> #<Issue number>
# ex) Resolve #122

##### COMMIT END #####
# <Type> 목록
# init: 초기화
# feat: 새로운 기능 추가
# update: 기능 보완 (업그레이드)
# fix: 버그 수정
# build: 빌드 관련 파일 수정
# ci: CI관련 설정 수정
# docs: 문서 (추가, 수정, 삭제)
# style: 스타일 (코드 형식, 세미콜론 추가: 로직에 변경 없는 경우)
# refactor: 코드 리팩토링
# test: 테스트 (테스트 코드 추가, 수정, 삭제: 로직에 변경 없는 경우)
# chore: 기타 변경사항 (빌드 혹은 패키지 매니저 수정사항)
##### 주 의 사 항 #####
# 제목 첫 글자는 대문자
# 제목은 명령문
# 제목 끝에 마침표(.) 금지
# 제목과 본문을 한 줄 띄워 분리
# 본문은 "어떻게" 보다 "무엇을"과 "왜"를 설명
# 본문에 여러 줄의 메시지를 작성할 땐 "-" 혹은 "번호"로 구분
